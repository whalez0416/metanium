# 메타니움(Metanium) 주요 기능 명세서

메타니움은 메타(Meta) 광고 대행 및 운영 업무를 자동화하고, 데이터 기반의 의사결정을 실시간으로 지원하기 위해 개발되었습니다.

---

## 1. 실시간 광고 성과 데이터 수집 (Data Fetching)
- **Meta Ads API 연동**: 광고 계정 내 모든 캠페인, 광고 세트, 소재의 성과 데이터를 실시간으로 수집합니다.
- **주요 수집 지표**: 지출액(Spend), 노출수(Impressions), 클릭수(Clicks), CTR, 리드(Leads), CPA 등.
- **상세 정보 추적**: 광고 생성 시점(created_at) 및 누적 지출액(lifetime_spend)을 추적하여 정교한 분석의 기초 데이터로 활용합니다.

## 2. 플레이북 기반 자동 의사결정 엔진 (Rule Engine)
마케터의 운영 노하우를 코드로 구현하여 광고 상태를 자동으로 평가합니다.
- **신규 소재 보호 규칙**: 생성 후 72시간 이내 또는 지출액 50,000원 미만인 소재는 성과와 관계없이 성급한 중단을 방지(KEEP)합니다.
- **주말 유연성 규칙**: 상담 지연이 발생하는 주말(토, 일)에는 CPA가 평소보다 20~30% 상승해도 즉시 중단하지 않고 유지(MAINTAIN)합니다.
- **지표 기반 스케일링**: 타겟 CPA 대비 성과가 탁월한 소재에 대해 즉각적인 예산 증액(SCALE UP)을 제안합니다.

## 3. 캠페인별 A/B 테스트 분석 (Comparative Analysis)
개별 광고의 절대적인 수치뿐만 아니라 캠페인 내에서의 상대적 경쟁력을 분석합니다.
- **캠페인 내부 그룹화**: 동일 캠페인 내 운영되는 여러 소재를 한데 묶어 평균 성과를 산출합니다.
- **승자/패자 판별**: 캠페인 평균 대비 CPA가 현저히 높은 '패배 소재'를 식별하여 중단을 제안(PAUSE)하고, '기회 소재'에는 예산 집중을 유도합니다.
- **예산 낭비(Waste) 방지**: 전환 성과 없이 과도한 지출이 발생하는 소재를 감지하여 조기에 차단합니다.

## 4. 조치 이력 및 제안 로깅 (Decision History)
- **제안 사항 자동 기록**: AI가 제안한 모든 조치(중단, 증액 등)를 `decision_history.json`에 `PENDING` 상태로 자동 기록합니다.
- **의사결정 프로세스**: 마케터는 기록된 제안을 검토하고 최종 승인(REJECT/APPROVE) 여부를 결정하여 운영 효율을 높일 수 있습니다.

---

## 5. 빠른 검증 시스템 (Verification & Demo)
- **시뮬레이션 모드**: 실제 API 연동 전에도 다양한 광고 운영 시나리오를 테스트할 수 있는 `demo.py`를 제공합니다.
- **가독성 높은 결과 출력**: 터미널 환경에서도 표 형식의 깔끔한 성과 요약 및 AI 제안 사항을 확인할 수 있습니다.
